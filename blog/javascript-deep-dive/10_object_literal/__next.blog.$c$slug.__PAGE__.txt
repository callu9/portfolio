2:T307d,<h1>10. 객체 리터럴</h1>
<h2>10.1. 객체란?</h2>
<p>자바스크립트는 객체(object)기반의 프로그래밍 언어이며, 원시 값을 제외한 나머지 값(함수, 배열, 정규 표현식 등)은 모두 객체다.</p>
<ul>
<li>원시 값과 객체의 비교
<ul>
<li>원시 타입은 단 하나의 값만 나타내지만, 객체 타입(object/reference type)은 다양한 타입의 값(원시 값 또는 다른 객체)를 하나의 단위로 구성한 복합적인 자료구조(data structure)</li>
<li>원시 타입의 값, 즉 원시 값은 변경 불가능한 값(immutable value)이지만 
객체 타입의 값, 즉 객체는 변경 가능한 값(mutabel value)이다.</li>
<li><a href="./11_comparing_primitive_value_and_object.md">11. 원시 값과 객체의 비교</a> 참고</li>
</ul>
</li>
</ul>
<h3>객체</h3>
<ul>
<li>객체 (object): 0개 이상의 프로퍼티로 구성된 집합
<ul>
<li>프로퍼티는 키(key)와 값(value)로 구성된다.</li>
<li>자바스크립트에서 사용할 수 있는 모든 값은 프로퍼티 값이 될 수 있으며, 함수는 일급 객체이므로 값으로 취급할 수 있다.</li>
<li>프로퍼티 값이 함수일 경우, 일반 함수와 구분하기 위해 <strong>메서드</strong>라 부른다.</li>
</ul>
</li>
<li>객체의 집합으로 프로그램을 표현하려는 프로그래밍 패러다임을 객체지향 프로그래밍이라 한다. (<a href="./19_prototype.md#191-%EA%B0%9D%EC%B2%B4%EC%A7%80%ED%96%A5-%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D">19.1. 객체지향 프로그래밍</a> 참고)</li>
</ul>
<h4>프로퍼티와 메서드의 역할</h4>
<p>객체는 프로퍼티와 메서드로 구성된 집합체로, 상태와 동작을 하나의 단위로 구조화할 수 있어 유용하다.</p>
<ul>
<li>프로퍼티(property): 객체의 상태를 나타내는 값(data)</li>
<li>메서드(method): 프로퍼티(상태 데이터)를 참조하고 조작할 수 있는 동작(behavior)</li>
</ul>
<h2>10.2. 객체 리터럴에 의한 객체 생성</h2>
<ul>
<li>
<p>클래스 기반 객체지향 언어(ex. C++, Java)는 클래스를 사전에 정의하고 필요한 시점에 <code>new</code> 연산자와 함께 생성자(constructor)를 호출하여 인스턴스를 생성하는 방식으로 객체를 생성한다.</p>
<ul>
<li>인스턴스(instance): 클래스에 의해 생성되어 메모리에 저장된 실체.</li>
<li>객체지향 프로그래밍에서 객체는 클래스와 인스턴스를 포함한 개념</li>
<li>클래스는 인스턴스를 생성하기 위한 템플릿의 역할</li>
<li>인스턴스는 객체가 메모리에 저장되어 실제로 존재하는 것에 초점을 맞춘 용어</li>
</ul>
</li>
<li>
<p>프로토타입 기반 객체지향 언어(ex. Javascript)는 다양한 객체 생성 방법을 지원한다.</p>
<ul>
<li>객체 리터럴</li>
<li><code>Object</code> 생성자 함수</li>
<li>생성자 함수</li>
<li><code>Object.create</code> 메서드</li>
<li>클래스(ES6)</li>
</ul>
</li>
</ul>
<h3>객체 리터럴</h3>
<p>객체 리터럴(객체를 생성하기 위한 표기법)을 사용한 객체 생성은 자바스크립트의 유연함과 강력함을 대표하는 객체 생성 방식으로, 가장 일반적이고 간단한 방법이다. (객체 리터럴 외의 객체 생성 방식은 모두 함수를 사용해 객체를 생성한다.)</p>
<pre><code class="language-js">var person = {
  name: 'Lee',
  sayHello: function () {
    console.log(`Hello! My name is ${this.name}.`);
  }
};

console.log(typeof person); // object
console.log(person); // {name: "Lee", sayHello: ƒ}
</code></pre>
<p>중괄호(<code>{...}</code>) 내에 0개 이상의 프로퍼티를 정의한다. 변수에 할당되는 시점에 자바스크립트 엔진은 객체 리터럴을 해석해 객체를 생성한다.</p>
<ul>
<li>만약 중괄호 내에 프로퍼티를 정의하지 않으면 빈 객체가 생성된다.</li>
<li>객체 리터럴의 중괄호는 코드 블록을 의미하지 않으며, 객체 리터럴은 값으로 평가되는 표현식이기 때문에 닫는 중괄호 뒤에는 세미콜론을 붙인다.</li>
<li>숫자값이나 문자열을 만드는 것과 유사하게 리터럴로 객체를 생성하며, 객체 리터럴에 프로퍼티를 포함시켜 객체를 생성함과 동시에 프로퍼티를 만들 수도 있고, 객체를 생성한 이후에 프로퍼티를 동적으로 추가할 수도 있다.</li>
</ul>
<h2>10.3. 프로퍼티</h2>
<p>객체는 프로퍼티의 집합이며, 프로퍼티는 키와 값으로 구성된다.</p>
<ul>
<li>프로퍼티 키와 프로퍼티 값으로 사용할 수 있는 값
<ul>
<li>프로퍼티 키: 빈 문자열을 포함하는 모든 문자열 또는 심벌값 
<ul>
<li>식별자 네이밍 규칙을 준수하는 프로퍼티 키를 사용할 것을 권장</li>
<li>빈 문자열도 프로퍼티 키로 사용할 수 있다.</li>
<li>문자열이나 심벌값 이외의 값을 사용하면 암묵적 타입 변환을 통해 문자열로 변환</li>
</ul>
</li>
<li>프로퍼티 값: 자바스크립트에서 사용할 수 있는 모든 값</li>
</ul>
</li>
</ul>
<p>프로퍼티를 나열할 때는 쉼표(,)로 구분한다.</p>
<pre><code class="language-js">var person = {
  // 프로퍼티 키는 name, 프로퍼티 값은 'Lee'
  name: 'Lee',
  // 프로퍼티 키는 age, 프로퍼티 값은 20
  age: 20
};
</code></pre>
<p>문자열 또는 문자열로 평가할 수 있는 표현식을 사용하여 프로퍼티 키를 동적으로 생성할 수도 있다. 프로퍼티 키로 사용할 표현식을 대괄호(<code>[...]</code>)로 묶어야 한다.</p>
<pre><code class="language-js">var obj = {};
var key = 'hello';

// ES5: 프로퍼티 키 동적 생성
obj[key] = 'world';
// ES6: 계산된 프로퍼티 이름
// var obj = { [key]: 'world' };

console.log(obj); // {hello: "world"}
</code></pre>
<p>이미 존재하는 프로퍼티 키를 중복 선언하면 나중에 선언한 프로퍼티가 먼저 선언한 프로퍼티를 덮어쓴다.</p>
<pre><code class="language-js">var foo = {
  name: 'Lee',
  name: 'Kim'
};

console.log(foo); // {name: "Kim"}
</code></pre>
<h2>10.4. 메서드</h2>
<p>프로퍼티 값이 함수일 경우 일반 함수와 구분하기 위해 메서드(method)라 부른다.</p>
<p>즉, 메서드는 객체에 묶여있는 함수를 의미한다.</p>
<pre><code class="language-js">var circle = {
  radius: 5, // ← 프로퍼티

  // 원의 지름
  getDiameter: function () { // ← 메서드
    return 2 * this.radius; // this는 circle을 가리킨다.
  }
};

console.log(circle.getDiameter()); // 10
</code></pre>
<h2>10.5. 프로퍼티 접근</h2>
<ul>
<li>마침표 프로퍼티 접근 연산자(<code>.</code>)를 사용하는 마침표 표기법 (dot notation)</li>
<li>대괄호 프로퍼티 접근 연산자(<code>[...]</code>)를 사용하는 대괄호 표기법 (bracket notation)
<ul>
<li>대괄호 프로퍼티 접근 연산자 내부에 지정하는 프로퍼티 키는 반드시 따옴표로 감싼 문자열이어야 한다.</li>
</ul>
</li>
</ul>
<pre><code class="language-js">var person = {
  name: 'Lee'
};

// 따옴표로 감싸지 않은 이름을 프로퍼티 키로 사용하면
// 자바스크립트 엔진이 식별자로 해석하기 때문에 에러가 발생
console.log(person[name]); // ReferenceError: name is not defined
</code></pre>
<p>객체에 존재하지 않는 프로퍼티에 접근하면 <code>undefined</code>를 반환한다.</p>
<pre><code class="language-js">var person = {
  name: 'Lee'
};

console.log(person.age); // undefined
</code></pre>
<p>브라우저 환경과 Node.js 환경에서의 실행결과 차이</p>
<ul>
<li>브라우저 환경에서는 <code>name</code>이라는 전역 변수(전역 객체 <code>window</code>의 프로퍼티)가 암묵적으로 존재
<ul>
<li><a href="./21_built_in_object.md#214-%EC%A0%84%EC%97%AD%EA%B0%9D%EC%B2%B4">21.4. 전역객체</a> 참고</li>
</ul>
</li>
<li>Node.js 환경에서는 어디에서도 <code>name</code>이라는 식별자(변수, 함수 등의 이름) 선언이 없으므로 <code>ReferenceError: name is not defined</code> 에러</li>
</ul>
<pre><code class="language-js">var person = {
  'last-name': 'Lee',
  1: 10
};

person.'last-name';  // -> SyntaxError: Unexpected string
person.last-name;    // -> 브라우저 환경: NaN
                     // -> Node.js 환경: ReferenceError: name is not defined
person[last-name];   // -> ReferenceError: last is not defined
person['last-name']; // -> Lee

// 프로퍼티 키가 숫자로 이뤄진 문자열인 경우 따옴표를 생략할 수 있다.
person.1;     // -> SyntaxError: Unexpected number
person.'1';   // -> SyntaxError: Unexpected string
person[1];    // -> 10 : person[1] -> person['1']
person['1'];  // -> 10
</code></pre>
<h2>10.6. 프로퍼티 값 갱신</h2>
<p>이미 존재하는 프로퍼티에 값을 할당하면 프로퍼티 값이 갱신된다.</p>
<pre><code class="language-js">var person = {
  name: 'Lee'
};

// person 객체에 name 프로퍼티가 존재하므로 name 프로퍼티의 값이 갱신된다.
person.name = 'Kim';

console.log(person);  // {name: "Kim"}
</code></pre>
<h2>10.7. 프로퍼티 동적 생성</h2>
<p>존재하지 않는 프로퍼티에 값을 할당하면 프로퍼티가 동적으로 생성되어 추가되고 프로퍼티 값이 할당된다.</p>
<pre><code class="language-js">var person = {
  name: 'Lee'
};

// person 객체에는 age 프로퍼티가 존재하지 않는다.
// 따라서 person 객체에 age 프로퍼티가 동적으로 생성되고 값이 할당된다.
person.age = 20;

console.log(person); // {name: "Lee", age: 20}
</code></pre>
<h2>10.8. 프로퍼티 삭제</h2>
<p><code>delete</code> 연산자는 객체의 프로퍼티를 삭제한다.</p>
<ul>
<li><code>delete</code> 연산자의 피연산자는 프로퍼티 값에 접근할 수 있는 표현식이어야 한다.</li>
<li>존재하지 않는 프로퍼티를 삭제하면 아무런 에러 없이 무시된다.</li>
</ul>
<pre><code class="language-js">var person = {
  name: 'Lee'
};

// 프로퍼티 동적 생성
person.age = 20;

// person 객체에 age 프로퍼티가 존재한다.
// 따라서 delete 연산자로 age 프로퍼티를 삭제할 수 있다.
delete person.age;

// person 객체에 address 프로퍼티가 존재하지 않는다.
// 따라서 delete 연산자로 address 프로퍼티를 삭제할 수 없다. 이때 에러가 발생하지 않는다.
delete person.address;

console.log(person); // {name: "Lee"}
</code></pre>
<h2>10.9. ES6에서 추가된 객체 리터럴의 확장 기능</h2>
<h3>10.9.1. 프로퍼티 축약 표현</h3>
<p>ES6에서는 프로퍼티 값으로 변수를 사용하는 경우 변수 이름과 프로퍼티 키가 동일한 이름일 때 프로퍼티 키를 생략(property shorthand)할 수 있다. 이때 프로퍼티 키는 변수 이름으로 자동 생성된다.</p>
<pre><code class="language-js">let x = 1, y = 2;

// 프로퍼티 축약 표현
const obj = { x, y };

console.log(obj); // {x: 1, y: 2}
</code></pre>
<h3>10.9.2. 계산된 프로퍼티 이름</h3>
<p>문자열 또는 문자열 타입 변환할 수 있는 값으로 평가되는 표현식을 사용해 프로퍼티 키를 동적으로 생성할 수 있다. 단, 프로퍼티 키로 사용할 표현식을 대괄호(<code>[...]</code>)로 묶어야 하며, 이를 계산된 프로퍼티 이름(computed property name)이라 한다.</p>
<pre><code class="language-js">const prefix = 'prop';
let i = 0;

// 객체 리터럴 내부에서 계산된 프로퍼티 이름으로 프로퍼티 키 동적 생성
const obj = {
  [`${prefix}-${++i}`]: i,
  [`${prefix}-${++i}`]: i,
  [`${prefix}-${++i}`]: i
};

console.log(obj); // {prop-1: 1, prop-2: 2, prop-3: 3}
</code></pre>
<h3>10.9.3. 메서드 축약 표현</h3>
<p><code>function</code> 키워드를 생략한 축약 표현을 사용할 수 있다.</p>
<pre><code class="language-js">const obj = {
  name: 'Lee',
  // 메서드 축약 표현
  sayHi() {
    console.log('Hi! ' + this.name);
  }
};

obj.sayHi(); // Hi! Lee
</code></pre>
<p>ES6의 메서드 축약 표현으로 정의한 메서드는 프로퍼티에 할당한 함수와 다르게 동작한다. (<a href="./26_ES6_additional_function_feature.md#262-%EB%A9%94%EC%86%8C%EB%93%9C">26.2. 메서드</a> 참고)</p>
