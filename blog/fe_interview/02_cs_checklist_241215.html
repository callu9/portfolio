3. 해시맵(HashMap)과 트리(Tree)의 주요 차이점은 무엇인가요?

| 자료구조 | 특징 | 장점 | 단점 | | --- | --- | --- | --- | | 해시맵 | 키-값 쌍으로 데이터를 저장한다. | 해시 함수를 사용한 검색, 삽입, 삭제가 평균적으로 O(1)로 가능하다. | 데이터의 순서가 보장되지 않으며, 해시 충돌 관리가 필요하다. | | 트리 | 계층적 구조로 데이터를 저장한다.(ex. 이진 탐색 트리(BST)는 정렬된 순서를 유지한다.) | 범위 검색이나 순회(traversal)에 유리하며, 평균 O(log n)의 시간 복잡도를 가진다. | 불균형이 발생하면 최악의 경우 O(n)이 될 수 있다. |

  • HashMap과 Map의 차이점은? ❓

    • JS에서는 Object와 Map이 HashMap의 역할
      • Java에서는 HashMap이 Map 인터페이스(키와 값 쌍으로 데이터를 저장하는 구조)를 구현한 클래스이며, 키의 중복을 허용하지 않지만 값의 중복은 허용한다.

  • 해시(Hash)란? ❓

    • 설명: 데이터를 다루는 기법으로, 키-값으로 매핑되는 과정 자체를 해싱(Hashing)이라 한다.
    • 특징:
      • 검색과 저장이 아주 빠르게 진행된다. 평균 시간 복잡도가 O(1)에 수렴 (key-value가 쌍으로 존재하며, key값이 인덱스로 변환되기 때문에)
      • 무결성: 같은 입력값에 대해 같은 출력값을 보장하며, 해시는 특정한 데이터를 상징하는 고정된 길이의 데이터로 변환하기 때문에, 원래의 데이터가 조금만 달라져도 확연하게 달라지는 특성을 가지고 있어 무결성을 지키는데 많은 도움을 준다.
      • 보안성: 복호화가 불가능한 단방향 암호화 기법의 특징을 가지고 있으며, 상대적으로 시스템 자원(CPU, 메모리) 소모가 덜하다.
    • 해시 값이란? 다양한 길이를 가진 데이터를 고정된 길이를 가진 데이터로 매핑한 값
    • Hash function이란? 임의의 길이의 데이터를 고정된 길이의 데이터로 매핑하는 단방향성 함수/알고리즘이다. 알고리즘 예시로 MD5, SHA 함수 등이 있으며, 사용 예시로 세션 키 도출, 일회용 패스워드(OTP) 생성, 주요 정보의 무결성 확인 등이 있다.

  • 해시 충돌(Hash Collision)이란? 서로 다른 문자열이 Hash function을 통해 Hash한 결과가 같은 경우(중복되는 경우)를 말하며, 저장할 버킷이 중복되는 현상이라고도 볼 수 있다.

    • 문제점: 충돌이 많아질 수록 같은 해시값의 데이터들에서 찾아야 할 데이터에 대한 탐색 과정이 추가되어 시간 복잡도가 O(1)에서 O(n)에 가까워질 수 있다. (충돌이 적을 수록 좋은 해시 함수가 된다.)
    • 해결방안:
      • Chaining 기법: 같은 해시 값을 갖는 데이터를 쇠사슬(chain) 모양으로 연결리스트에서 연결하여 관리하는 방법
      • Open Addressing 기법: 빈 버킷을 찾을 때까지 해시를 반복하여 관리하는 방법

Set은 ES6에서 새로 추가된 데이터 타입으로, 유일한 값을 저장하는 자료형

const array = [1, 2, 2, 3, 3, 3, 4, 4, 4, 4];

  1. filter()와 indexOf() 사용
const array = [1, 2, 2, 3, 3, 3, 4, 4, 4, 4];
const uniqueArray = array.filter((value, idx, arr) => arr.indexOf(value) === idx);
  1. reduce() 사용
const array = [1, 2, 2, 3, 3, 3, 4, 4, 4, 4];
const uniqueArray = array.reduce((unique, item) => {
	return unique.includes(item) ? unique : [...unique, item];
}, []);

3. 브라우저에서 URL을 입력하고 Enter를 누르면 발생하는 일련의 과정을 설명하세요.

  1. DNS 조회: 도메인 이름을 IP 주소로 변환
  2. TCP 연결: 서버와의 연결을 수립 (HTTPS의 경우 TLS 핸드쉐이크 포함)
  3. HTTP 요청: 브라우저가 서버에 요청을 보냄
  4. 서버 응답: 서버가 요청에 대한 응답을 보냄 (HTML, CSS, JS 등의 리소스 포함)
  5. 렌더링 (CRP, Critical Rendering Path)
    • 브라우저가 HTML을 파싱하여 DOM 생성, CSS 파싱하여 CSSOM 생성
    • DOM + CSSOM 결합하여 렌더트리 형성
    • 레이아웃
    • 페인트

4. DNS(Domain Name System)란 무엇이며, DNS의 역할은 무엇인가요?

  • 설명: 도메인 이름을 IP 주소로 변환하여 주는 시스템을 말하며, 도메인 이름 시스템 레코드는 DNS 서버 내에서 도메인 이름을 인터넷 프로토콜(IP) 주소와 연결하는 데 사용되는 일련의 지침이다.
  • 역할: 도메인 이름을 IP 주소로 변환하여, 클라이언트가 올바른 서버에 연결할 수 있도록 돕는 역할을 한다.

5. REST API란 무엇이며, 장단점은 무엇인가요?

  • REST: 리소스 기반의 API 설계를 뜻한다. URL마다 고정된 데이터 구조를 가지고 있으며, 여러 엔드 포인트를 사용한다.
  • 장점: 단순하고 캐싱, HTTP 메서드 활용 쉬움
  • 단점: 필요한 데이터 이상을 가져올 수 있어 과다 응답(over-fetching) 문제가 발생할 수 있음
