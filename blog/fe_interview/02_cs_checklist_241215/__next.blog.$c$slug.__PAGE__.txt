<p>| 자료구조	| 특징 | 장점 | 단점 |

| 해시맵 | 키-값 쌍으로 데이터를 저장한다. | 해시 함수를 사용한 검색, 삽입, 삭제가 평균적으로 O(1)로 가능하다. | 데이터의 순서가 보장되지 않으며, 해시 충돌 관리가 필요하다. |
| 트리 | 계층적 구조로 데이터를 저장한다.(ex. 이진 탐색 트리(BST)는 정렬된 순서를 유지한다.) | 범위 검색이나 순회(traversal)에 유리하며, 평균 O(log n)의 시간 복잡도를 가진다. | 불균형이 발생하면 최악의 경우 O(n)이 될 수 있다. |</p>
<ul>
<li>
<p>HashMap과 Map의 차이점은? ❓</p>
<ul>
<li>JS에서는 Object와 Map이 HashMap의 역할
<ul>
<li>Java에서는 HashMap이 Map 인터페이스(키와 값 쌍으로 데이터를 저장하는 구조)를 구현한 클래스이며, 키의 중복을 허용하지 않지만 값의 중복은 허용한다.</li>
</ul>
</li>
</ul>
</li>
<li>
<p>해시(Hash)란? ❓</p>
<ul>
<li>설명: 데이터를 다루는 기법으로, 키-값으로 매핑되는 과정 자체를 해싱(Hashing)이라 한다.</li>
<li>특징:
<ul>
<li>검색과 저장이 아주 빠르게 진행된다. 평균 시간 복잡도가 O(1)에 수렴 (key-value가 쌍으로 존재하며, key값이 인덱스로 변환되기 때문에)</li>
<li>무결성: 같은 입력값에 대해 같은 출력값을 보장하며, 해시는 특정한 데이터를 상징하는 고정된 길이의 데이터로 변환하기 때문에, 원래의 데이터가 조금만 달라져도 확연하게 달라지는 특성을 가지고 있어 무결성을 지키는데 많은 도움을 준다.</li>
<li>보안성: 복호화가 불가능한 단방향 암호화 기법의 특징을 가지고 있으며, 상대적으로 시스템 자원(CPU, 메모리) 소모가 덜하다.</li>
</ul>
</li>
<li>해시 값이란? 다양한 길이를 가진 데이터를 <strong>고정된 길이를 가진 데이터로 매핑</strong>한 값</li>
<li>Hash function이란? 임의의 길이의 데이터를 고정된 길이의 데이터로 매핑하는 단방향성 함수/알고리즘이다. 알고리즘 예시로 MD5, SHA 함수 등이 있으며, 사용 예시로 세션 키 도출, 일회용 패스워드(OTP) 생성, 주요 정보의 무결성 확인 등이 있다.</li>
</ul>
</li>
<li>
<p>해시 충돌(Hash Collision)이란? 서로 다른 문자열이 Hash function을 통해 Hash한 결과가 같은 경우(중복되는 경우)를 말하며, 저장할 버킷이 중복되는 현상이라고도 볼 수 있다.</p>
<ul>
<li>문제점: 충돌이 많아질 수록 같은 해시값의 데이터들에서 찾아야 할 데이터에 대한 탐색 과정이 추가되어 시간 복잡도가 O(1)에서 O(n)에 가까워질 수 있다. (충돌이 적을 수록 좋은 해시 함수가 된다.)</li>
<li>해결방안:
<ul>
<li>Chaining 기법: 같은 해시 값을 갖는 데이터를 쇠사슬(chain) 모양으로 연결리스트에서 연결하여 관리하는 방법</li>
<li>Open Addressing 기법: 빈 버킷을 찾을 때까지 해시를 반복하여 관리하는 방법</li>
</ul>
</li>
</ul>
</li>
</ul>
