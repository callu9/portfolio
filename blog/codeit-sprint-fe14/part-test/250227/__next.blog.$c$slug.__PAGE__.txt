<h2>3. 브라우저 또는 Node.js 환경에서 비동기 함수의 콜백이 실행되는 원리에 대해 setTimeout 을 예로 들어 설명해 주세요.</h2>
<p>https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Execution_model</p>
<p>브라우저 또는 Node.js 환경, 특히 싱글스레드 환경에서의 비동기 함수의 콜백은 즉시 실행되지 않고, 다른 작업이 모두 완료된 후 실행이 됩니다.
예를 들어, setTimeout이 Call Stack에 호출되면, Web API에 타이머를 설정한 후, Call Stack에서 제거됩니다. Call Stack의 모든 작업이 완료되면, Callback Queue에 들어 있던 setTimeout의 콜백 함수가 실행됩니다.
결론적으로 아무리 setTimeout을 먼저 호출했다 하더라도, Call Stack이 비어있지 않다면 setTimeout의 콜백 함수는 Call Stack의 작업이 모두 완료되기까지 기다린 후 호출됩니다.</p>
<h2>4. GET 요청과 POST 요청을 비교해 설명해 주세요.</h2>
<p>GET 요청은 자원을 조회할 때 사용합니다. '?keyword=코드잇&#x26;offset=1'과 같은 쿼리를 통해 다양한 정보를 담을 수 있으며, Request body는 사용하지 않습니다. 또한 멱등성을 지녀, 같은 조회 요청에 같은 응답을 보장받을 수 있습니다.
반면, POST 요청은 자원을 생성할 때 사용합니다. Request body를 사용해 자원 생성과 관련된 정보를 전달하며, 멱등성을 띠고 있지 않다고 볼 수 있습니다. 즉, 동일한 리퀘스트를 여러번 보내어 이미 저장된 정보라고 하더라도 계속해서 리소스 생성 과정을 거치기 때문에 동일한 응답을 받을 것을 보장하지 못합니다.</p>

<h2>6. Flexbox와 Grid의 차이점을 설명하고, 각각 어떤 경우에 사용하는게 좋다고 생각하나요?</h2>
<p>Flexbox는 열 또는 행을 통해 요소를 나열하기 위해 사용됩니다. 한 행, 또는 열에 요소를 나열하기 위해 각 요소에 flex-grow, flex-shrink 같은 속성을 주어 자동으로 늘어나거나 줄어들게 할 수도 있습니다. 주로 키워드(혹은 해시태그) 나열, 메뉴바(Nav) 등에 사용합니다.
Grid는 열과 행 둘 다 활용하여 요소의 배치를 위하여 사용됩니다. 주로 게시글 미리보기, 상품 목록, 피드 등 균일한 간격과 크기의 요소를 정갈하게 나열하기 위해 사용합니다.</p>
<h2>7. 동기 비동기 실행에 대해 설명해 주세요.</h2>
<p>동기와 비동기 실행의 특징(특정 작업을 다 처리하기 전에 다음 코드로 넘어가는지, 다 처리한 후에 다음 코드로 넘어가는지)에 대해 정확하게 작성</p>
<p>동기 실행이란, 순차적으로 작업이 실행되며 작업 완료 전에는 다른 작업을 수행하지 않습니다. 따라서 코드가 작성된 순서대로 결과가 나타납니다.
그러나 비동기 실행은 작업 요청 후 다음 작업을 즉시 실행하며, 이전 작업의 완료 여부와 상관 없이 다른 작업을 진행합니다. 단, 모든 작업이 완료된 시점에 콜백, Promise 등의 결과를 처리합니다. 따라서 코드가 작성된 순서대로 결과의 순서가 보장된다고 할 수 없습니다.</p>
<h2>8. 배열에 있는 filter, map과 같은 메소드에 있는 콜백과 setTimeout에 있는 콜백의 차이에 대해 설명해 주세요.</h2>
<p>메소드 내 콜백은 동기적으로 처리되며, 호출된 즉시 실행됩니다. 그러나 setTimeout의 콜백은 Callback Queue에 들어가게 되며, Call Stack이 비워진 후에 스택에 쌓여 실행되기 때문에 작업의 순서가 후순위가 됩니다.</p>
<h3>해설</h3>
<p>바스크립트에서 콜백(callback)이라는 개념은 함수가 다른 함수의 인자로 전달되어, 그 함수에 의해 호출되는 패턴을 의미합니다. 하지만 모든 콜백이 동일한 방식으로 작동하지는 않습니다. 특히 배열 메소드(filter, map 등)의 콜백과 비동기 함수(setTimeout 등)의 콜백은 서로 다른 방식으로 작동합니다.</p>
<p>동기(synchronous) 콜백
배열 메소드에서 사용되는 콜백 함수는 동기적(synchronous)으로 작동합니다.
동기적이라는 것은, 메소드가 호출되면 배열의 각 요소에 대해 순차적으로 콜백이 실행된 후 결과가 반환된다는 의미입니다.
이때 다음 코드는 콜백 함수가 모두 실행된 후에 진행됩니다.</p>
<p>비동기(asynchronous) 콜백
setTimeout과 같은 함수의 콜백은 비동기적(asynchronous)으로 작동합니다.
이 함수들은 자바스크립트의 이벤트 루프를 통해 나중에 실행되며, 다른 코드의 실행이 끝난 후에 호출됩니다.
지정된 시간 이후에 콜백이 실행되지만, 그 사이에 다음 코드가 먼저 실행됩니다.</p>
<h2>9. 자바스크립트에서 콜백(콜백 함수)이 무엇인지 설명해 주세요.</h2>
<p>콜백이란 다른 함수의 인자로 전달되어, 작업이 끝난 후 호출되는 함수를 말합니다. 자바스크립트에서 함수는 일급 객체이기 때문에 함수의 인자로 전달될 수도 있고 반환값으로도 사용될 수 있습니다. 콜백 함수는 비동기 작업에서 함수의 실행 순서를 제어하는 데 유용합니다.</p>
<h3>해설</h3>
<p>콜백 함수란 다른 함수의 인자로 전달되어 그 함수의 실행이 끝난 후 호출되는 함수를 말합니다. 자바스크립트에서 함수는 일급 객체이기 때문에, 변수에 할당하거나 인자로 전달할 수 있습니다. 이 특성을 활용해, 콜백 함수를 정의한 후 특정 시점에 실행되도록 할 수 있습니다.</p>
<p>function greet(name, callback) {
console.log(안녕하세요, ${name}!);
callback(); // 전달받은 콜백 함수 실행
}</p>
<p>function sayGoodbye() {
console.log("안녕히 가세요!");
}</p>
